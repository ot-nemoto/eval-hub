# requirements.md — 機能要件・非機能要件

## 機能要件

### 1. ユーザー・認証

- ユーザーはメールアドレス + パスワードでログインできる
- ロールは `admin` / `manager` / `member` の3種類
- `member` は自分のデータのみ閲覧・編集できる
- `manager` は担当メンバーのデータを閲覧・評価できる
- `admin` はすべてのマスタデータを管理できる

### 2. 社員プロフィール管理

- 社員の基本情報（氏名・所属事業部・入社日）を登録・編集できる
- 配属履歴（年・事業部）を年単位で記録できる
- 現在のロール（自己認識・公式）を複数選択で登録できる
- 現在の役職（なし / チームリーダー / EL / 課長 / 部長 / 取締役）を登録できる
- 社長面談の希望有無を設定できる

### 3. キャリアプラン管理

- 年度単位でキャリアプランを作成できる
- 将来ロール（3〜5年後）と将来役職を登録できる
- 年度目標を複数登録でき、各目標に対して以下を記録できる
  - 大項目 / 目標タイトル / Goal（達成基準）
  - 取り組み内容 / 対象期間（1Q〜4Q）
  - 進捗・結果（本人記入）
  - 関連する評価項目UID（複数紐付け可）
- 本年度の成果・自由PR欄を記述できる
- 来年度に向けた目標・コメントを記述できる

### 4. 面談コメント管理

- 中間面談コメント（上長記入）を登録できる
- 期末面談コメント（上長記入）を登録できる
- 面談コメントは年度・社員ごとに管理される

### 5. 評価管理

- 評価項目は UID（例: `1-1-1`）で一意に識別される
- 評価項目の構造：大分類 > 中分類 > 項目番号
- 各評価項目に対して以下を記録できる
  - 自己採点（なし / 可 / 良 / 優）
  - 自己採点理由（テキスト）
  - 上長採点（なし / 可 / 良 / 優）
  - 上長採点理由（テキスト）
- 評価は年度ごとに記録される
- ２年ルール（前年評価を引き継ぐ項目）フラグを管理できる

### 6. ロール管理

- ロールは分類（engineer / Architect / Manager等）と役割名で管理される
- 各ロールに認定基準（Required / Special）を定義できる
- ロールと評価項目のマッピングを管理できる
  - 必須度（○：必須 / △：半分以上必須）
  - 必要な評価水準（優 / 良 / 可）
  - 判定結果（qualified / unqualified / none / pass）
- ロール認定の合否（qualified / unqualified）を社員ごとに記録できる

### 7. 配点管理

- 評価項目ごとに事業部別の配点（重み）を設定できる
- 事業部は「勤怠 / ITE / 観光 / サ開 / 本部」など複数管理できる
- 配点の合計は各カテゴリ内で100点になるよう管理する

### 8. 月次実績管理

- プロダクト・タスク・月を軸に実績を記録できる（チェックボックス形式）
- 月は年度内の12ヶ月を管理する
- プロダクト名とタスク名を自由に登録できる

## 非機能要件

| 項目 | 要件 |
|---|---|
| 対応ブラウザ | Chrome / Edge 最新版 |
| レスポンシブ | PC（1280px以上）を優先、タブレット対応は任意 |
| 認証セッション | JWT または Cookie セッション（有効期限24時間） |
| パフォーマンス | 一覧画面の初期表示 2秒以内 |
| セキュリティ | HTTPS 必須、SQL インジェクション・XSS 対策 |
| データ保持 | 年度をまたいだ評価履歴を無期限保持 |

## ユースケース（主要フロー）

### UC-1: 自己評価の入力

1. メンバーがログイン
2. 評価一覧画面を開く
3. 評価項目を選択し、自己採点（可/良/優）と理由を入力
4. 保存する

### UC-2: 上長評価の入力

1. マネージャーがログイン
2. 担当メンバーを選択
3. 評価一覧画面で上長採点と理由を入力
4. 面談コメントを記録して保存する

### UC-3: ロール認定の確認

1. メンバーまたはマネージャーがロール一覧を開く
2. 各ロールの必要評価項目と現在の採点を照合
3. qualified / unqualified の判定結果を確認する

### UC-4: キャリアプランの年度更新

1. 年度切り替え時に管理者が新年度を作成
2. メンバーが新年度のキャリアプランに将来ロール・目標を入力
3. 上長が面談後にコメントを追記する
