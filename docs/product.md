# product.md — プロダクト概要

## プロダクト名

**EvalHub** — エンジニア評価・キャリアプラン管理システム

## 背景・目的

エンジニアの評価とキャリアプランを Excel で管理していた。
シートが肥大化し、更新・参照・集計が困難になっているため、Web アプリケーションとして再構築する。

## ターゲットユーザー

| ロール | 説明 |
|---|---|
| エンジニア（本人） | 自己評価の入力、目標・取り組みの記録、キャリアプランの設定 |
| 上長（EL・課長等） | 上長評価の入力、面談コメントの記録、メンバー一覧の参照 |
| 管理者（HR・事業部長） | 評価項目・ロール定義の管理、事業部別配点の設定 |

## 解決する課題

- Excel ファイルの属人管理・バージョン管理の煩雑さ
- 自己評価と上長評価の突合・集計の手間
- ロール認定状況（qualified / unqualified）の可視化不足
- 年度ごとのキャリアプランの変遷が追いにくい
- 月次実績（タスク記録）との評価の紐付けが困難

## 主要機能の全体像

```
EvalHub
├── キャリアプラン管理    ← Overview2025 / Overview2026 相当
│   ├── 基本情報（氏名・所属・入社日・配属履歴）
│   ├── 現在/将来のロール・役職
│   └── 年度目標・取り組み・進捗・面談コメント
│
├── 評価管理              ← evaluation シート相当
│   ├── 評価項目マスタ（UID・説明・評価基準）
│   ├── 自己評価入力（可/良/優 + コメント）
│   └── 上長評価入力（可/良/優 + コメント）
│
├── ロール管理            ← role / role-evaluation-Mapping シート相当
│   ├── ロール定義（認定基準・説明）
│   ├── ロールと評価項目のマッピング
│   └── ロール認定判定（qualified / unqualified / none）
│
├── 配点管理              ← Allocation シート相当
│   └── 事業部別・評価項目別の配点設定
│
└── 実績管理              ← 実績 シート相当
    └── 月次タスク実績の記録・参照
```

## スコープ外（初期リリース）

- 給与・報酬との連携
- 外部 HR システムとの統合
- モバイルアプリ
