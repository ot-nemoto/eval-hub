# product.md — プロダクト概要

## プロダクト名

**EvalHub** — エンジニア評価・キャリアプラン管理システム

## 背景・目的

エンジニアの評価とキャリアプランを Excel で管理していた。
シートが肥大化し、更新・参照・集計が困難になっているため、Web アプリケーションとして再構築する。

## ターゲットユーザー

| ロール | 説明 |
|---|---|
| メンバー（被評価者） | 自己評価の入力 |
| メンバー（評価者） | 担当する被評価者への評価入力（同一ユーザーが両方になれる） |
| 管理者（admin） | ユーザー管理・評価者アサイン・マスタデータ管理 |

> 「上長」という固定ロールは持たない。評価者/被評価者の関係は年度ごとに `evaluation_assignments` で管理する。

## 解決する課題

- Excel ファイルの属人管理・バージョン管理の煩雑さ
- 自己評価と上長評価の突合・集計の手間
- ロール認定状況（qualified / unqualified）の可視化不足
- 年度ごとのキャリアプランの変遷が追いにくい
- 月次実績（タスク記録）との評価の紐付けが困難

## 主要機能の全体像

```
EvalHub
├── キャリアプラン管理    ← Overview2025 / Overview2026 相当
│   ├── 基本情報（氏名・所属・入社日・配属履歴）
│   ├── 現在/将来のロール・役職
│   └── 年度目標・取り組み・進捗・面談コメント
│
├── 評価管理              ← evaluation シート相当
│   ├── 評価項目マスタ（UID・説明・評価基準）
│   ├── 自己評価入力（可/良/優 + コメント）
│   └── 上長評価入力（可/良/優 + コメント）
│
├── ロール管理            ← role / role-evaluation-Mapping シート相当
│   ├── ロール定義（認定基準・説明）
│   ├── ロールと評価項目のマッピング
│   └── ロール認定判定（qualified / unqualified / none）
│
├── 配点管理              ← Allocation シート相当
│   └── 事業部別・評価項目別の配点設定
│
└── 実績管理              ← 実績 シート相当
    └── 月次タスク実績の記録・参照
```

## 成功指標

| 指標 | 内容 |
|---|---|
| 自己評価完了率 | 全メンバーが自己評価を当該年度の期限内に入力できること |
| 評価者評価完了率 | 全評価者が担当する被評価者の評価を年度期限内に完了できること |
| 管理業務のシステム完結 | 管理者が評価マスタ・評価者アサイン・年度管理をシステム上で完結できること |
