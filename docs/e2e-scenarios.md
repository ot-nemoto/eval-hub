# e2e-scenarios.md — E2Eテストシナリオ

PlaywrightMCP が E2E テストを実施するためのシナリオ集。

---

## 報告フォーマット

各シナリオの結果を以下のステータスで報告する。

| ステータス | 意味 |
|----------|------|
| OK | 期待値通り |
| NG | 期待値と異なる（再現手順と実際の挙動を記載） |
| SKIP | 技術的制約等で実行不可（理由を記載） |

---

## 推奨実施順序

複数セッション・複数エージェントで実施する際の引き継ぎ漏れを防ぐため、以下の順序で進めることを推奨する。

| フェーズ | セクション | 主なロール | 備考 |
|---------|-----------|----------|------|
| 1 | 認証・リダイレクト | 未ログイン / 全ロール | 副作用なし |
| 2 | ナビゲーション | MEMBER / ADMIN | 副作用なし |
| 3 | プロフィール名変更 | MEMBER | 名前変更あり。最後に元の名前に戻す手順を含む |
| 4 | 自己評価 | MEMBER | 採点データを変更する。リロードで永続化を確認 |
| 5 | メンバー一覧 | MEMBER / ADMIN | 副作用なし |
| 6 | メンバー別評価 | MEMBER / ADMIN | スコア・コメントを変更する |
| 7 | 管理：評価一覧（マトリクス） | ADMIN | 副作用なし |
| 8 | 管理：評価進捗ダッシュボード | ADMIN | 副作用なし |
| 9 | 管理：評価者アサイン管理 | ADMIN | アサインデータを追加。**ユーザー管理 #9 の前に実施** |
| 10 | 管理：大分類・中分類・評価項目マスタ | ADMIN | テスト用データを追加・削除・並び替えする |
| 11 | 管理：ユーザー管理 | ADMIN | ロール変更・削除あり。**#9（torikawa 削除）は最後に実施** |
| 12 | 管理：年度管理 | ADMIN | 年度追加・削除・ロック変更あり。**最後に復元手順まで実施** |
| 13 | 管理：バージョン管理（マスタ管理ページ内） | ADMIN | バージョン保存・復元・削除。年度へのバージョン割り当て・解除 |
| 14 | ユーザー分離 | MEMBER / ADMIN | 副作用なし |

> **注意：** ユーザー管理 #9（torikawa 削除）の後、評価者アサイン管理で torikawa を使うシナリオは失敗する。評価者アサイン管理はユーザー管理 #9 の **前** に実施すること。
