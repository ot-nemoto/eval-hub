# tasks.md — 実装タスク・順序

## MVP スコープ

> **MVP = Phase 1 + Phase 2 + Phase 3 + 5-6（デプロイ）**
>
> MVP の定義：「社員が自己評価を入力し、上長が評価・コメントを記録し、ロール認定状況を確認できる」
>
> Phase 4（実績管理・配点管理）および Phase 5（UI整備）は v1.1 以降。

## フェーズ構成

```
【MVP】
Phase 1: 基盤構築
Phase 2: コアデータ管理（評価・ロール）
Phase 3: キャリアプラン・目標管理

【v1.1 以降】
Phase 4: 実績管理・集計
Phase 5: UI 整備・仕上げ（5-6 デプロイのみ MVP）
```
